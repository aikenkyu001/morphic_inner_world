## 🧪 The Pipeline: NL → AST → EVAL
To eliminate probabilistic uncertainty, we employ a multi-stage deterministic execution strategy:
1.  **NL → AST (Outer World & Bridge)**: Ambiguous NL specifications are filtered through a **Semantic Dictionary** and synthesized into a rigorous **Abstract Syntax Tree (AST)** using **Arity-based Stacking Logic**.
2.  **AST → EVAL (Inner World)**: The Morphic VM (a pure functional evaluator) reduces the AST to its **Normal Form**, calculating the result as a mathematical necessity.
3.  **Bilingual Integrity**: The pipeline ensures that both English and Japanese inputs map to the **exact same AST**, proving language-invariant logic.

## 🛠 Getting Started

### Installation
```bash
git clone https://github.com/aikenkyu001/morphic_inner_world.git
cd morphic_inner_world
# Activate your virtual environment
source venv/bin/activate
pip install -r requirements.txt
```

### Running the Full Benchmark
Rebuild all solutions from specifications and run the 100.0% success bilingual verification.
```bash
python3 Scripts/rebuild_solutions_bilingual.py
python3 Scripts/run_benchmark.py --bilingual
```

### Verifying Logical Integrity (Honest Cycle)
Prove the "NL-AST-NL" bridge by reconstructing original logic from generated code.
```bash
python3 Scripts/verify_all_honest_cycles_jp.py
python3 Scripts/verify_all_honest_cycles_en.py
```
