# 幾何学的自然言語仕様プロトコル (Geometric NL Specification Protocol)

本ドキュメントは、外界の意図を内界の論理へと決定論的に写像するための、自然言語記述における厳格な規約を定義する。

## 1. 言語の制限
- **辞書の遵守:** 論理フロー（LOGIC_FLOW）内で使用する動作語は、必ず `Theory/semantic_dictionary.json` に定義されたフレーズを使用しなければならない。
- **体言止めの推奨 (日本語):** 冗長な語尾（〜する、〜を求める）を排し、名詞句または基本形で記述すること。

## 2. セクション構造
仕様書は以下のセクションをこの順序で含まなければならない。

1. **TASK / タスク:** タスクの識別名。
2. **INTERFACE / インターフェース:** クラスメソッドのシグネチャ。
3. **LOGIC_FLOW / 論理フロー:** 
   - **INPUT / 入力:** 引数名のリスト。
   - **LET logic_result = / 定義 logic_result = :** 内界の論理（AST）を記述する。**※ 物理的制約: 必ず1行で記述すること。**
4. **OUTPUT / 出力:** 
   - 計算結果（res.value）をどのように呼び出し元に返すか、または副作用（インプレース書き換え）を行う Python コードを記述する。

## 3. 物理的制約
- **一行性の原則:** `LET logic_result` の右辺に改行を含めてはならない。複雑な論理はネストした `App` または `Let` として一行に凝縮すること。
- **インデントの維持:** `OUTPUT` セクションに記述する Python コードは、適切なインデントを維持すること。シンセサイザーは最初の非空行のインデントを基準に `dedent` を実行する。

## 4. 記述例 (活動選択問題)
```text
# [幾何学的NL仕様]
タスク: activity_selection
インターフェース: maxActivities(self, a)

論理フロー:
  入力: a
  LET logic_result = App(Var('filter_overlapping'), App(Var('sort_by_end'), Var('a')))
  出力: return len(res.value)
```
