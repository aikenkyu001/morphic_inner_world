# 幾何学的自然言語仕様プロトコル (Geometric NL Specification Protocol) v1.2

本ドキュメントは、外界の意図を内界の論理へと決定論的に写像するための、自然言語記述における厳格な規約を定義する。

## 1. 基本規約
- **辞書の遵守:** 論理フロー（LOGIC / 論理フロー）内で使用する動作語は、必ず `Theory/semantic_dictionary.json` に定義されたフレーズを使用しなければならない。
- **番号付け:** 論理ステップは必ず `1. `, `2. ` といった形式でナンバリングされなければならない。
- **純粋性の維持:** 論理フロー内には、計算に無関係な主観的表現や形容詞を含めてはならない。

## 2. 構造定義
仕様ファイル (`.nl`) は以下のブロックで構成される。

### ヘッダー
- `タスク:` / `TASK:` : タスクの一意識別子。
- `インターフェース:` / `INTERFACE:` : メソッド名と引数。例: `solve(self, arg1)`

### 論理ブロック
- `入力:` / `INPUT:` : 処理対象の変数名。
- `論理フロー:` / `LOGIC:` : ナンバリングされた実行ステップ。
- `出力:` / `OUTPUT:` : 戻り値の定義。

## 3. 複数インターフェースの記述
1つのタスクで複数の機能（例: 構造の直列化と復元）を定義する場合、各インターフェース定義ブロックを **`---` (ハイフン3つ)** で区切らなければならない。

```text
タスク: my_task
インターフェース: func1(arg1)
...
---
インターフェース: func2(arg1)
...
```

## 4. スケール不変性の保証
- **v8000 (文脈長)**: 論理フローブロックの前後には、任意の量の充填テキスト（ノイズ）を含めることができる。シンセサイザーはナンバリングされた行のみを抽出し、幾何学的射影を行う。
- **d15 (論理深度)**: ステップを多重化することで、AST の深い入れ子構造を構築する。
